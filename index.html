Introduction

  • Definition

    Software systems engineering is the creation of new systems in the world that arises from the development of design and coding. The concept of software systems engineering is changing but in general the aim is to make new software in order to revolutionize the digital world. Currently, there is a plethora of software systems that are constantly advancing due to the elixir of knowledge obtained by developers who aim to push the boundaries of software engineering.

    Scope of research area

    In order to solve real world problems, systems of software are evolving, so it covers a lot of subjects. However, there are several areas are tightly intertwined with it: requirements engineering, software processes, software architecture, software analysis (requirement analysis), testing and software deployment

  • Enbedded System

    An embedded system is a computer system which is embedded as part of a complete device. There are a lot of examples in daily life like cookers, cellphone, cars…etc. Embedded systems are designed to do some specific task, rather than be a general-purpose computer for multiple tasks, for example, the system in our cellphones aim to help us use cellphone easily but not do complicated programming or calculating.

  • Open-source software (OSS)

    Open-source software (OSS) is computer software with it source code made available with a license in which the copyright provides the right to study, change, and distribute the software to anyone and for any purpose. Future

  • The future of software engineering is probably the hardest thing to predict, however there are definitely some predictions we can make as to what we can expect. We can expect software engineers to become even more efficient with their code resulting in the computing speed of programs to be quicker. This means the future possibly holds a scope where we the users hardly have to wait whilst a program is being run.

    Imagine in a few years these things happen:

  • - A government API to calculate taxes on all online purchases at any locations was made.

  • - A distributed traffic regulation system for a network of self-driving cars and drones.

  • - A planetary asteroid deflector system in response to an asteroid mining operation gone wrong.

  • The creation of any of these softwares would revolutionise the digital world and in some cases, make our lives a lot easier too. The future of software engineering truly lies in the types of creations that take place between now and the next 50 years. These can impact the digital world on the massive scale and allow us to lift software engineering to the next level to where perhaps softwares are able to create new softwares in a matter of minutes.

    The future also lies in the basic programming skills that people possess. In the future, pretty much everybody will be familiar with some sort of programming whether that be excel sheets or complicated python programs. Software engineering will reach a level where dreams such as self-driving vehicles will be made possible due to the advancement in the field especially from the vision and graphics aid.

    In terms of software engineering as a career, the field seems to be very promising especially because of the rise of automated systems. Due to this the number of programmers needed has been increasing and will continue to follow this pattern due to everything slowly becoming digitalised. Even money is becoming digitalised e.g. the invention of the bitcoin. Rapid inventions such as these result in more software engineers being needed in order to create the new softwares needed to allow for these new inventions to run.
